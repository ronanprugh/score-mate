# Task 03 Proofs — Snapshot team/league catalog + new in-memory search route

## Task Summary

This task proves the favorites typeahead is now powered by a committed JSON catalog of every team in every supported ESPN league, with the route handler doing a pure in-memory substring search. No upstream calls happen at request time, no cold-cache latency cliff after deploy, and the search is fully deterministic in tests. The catalog is regenerated by an operator-run script (`scripts/refresh-espn-catalog.ts`) that's documented for quarterly cadence; it supports `--dry-run` so an operator can preview the plan before fetching.

The cleanup absorbed here (T3.9) also lands the final removal of `lib/sportsdb/`, the `Sport` union drop of `"Tennis"`, and the corresponding test-fixture migrations to ESPN id shapes.

## What This Task Proves

- `scripts/refresh-espn-catalog.ts` runs against the live ESPN API and produces a deterministically-sorted JSON catalog. The `--dry-run` flag lists the plan without writing.
- `lib/espn/catalog.json` contains **1,675 teams across 19 leagues**, well above the spec's ≥500 threshold (per-league breakdown in [03-catalog-counts.md](./03-catalog-counts.md)).
- The committed catalog covers exactly the three v1 sports — Soccer, American Football, Basketball — with no Tennis residue.
- `lib/espn/catalog.ts` exposes `searchCatalogTeams` and `searchCatalogLeagues` with case-insensitive substring + optional sport filter.
- `app/api/favorites/search/route.ts` is rewritten to call the new catalog helpers (plus the existing events-catalog and sport-name matching) with the same `{ type, externalId, displayName, sport, metadata? }` response contract, the same per-category cap of 10, and the same result order (sport, event, league, team).
- All eight Success Metric §4 breadth queries (`arsenal`, `lakers`, `chiefs`, `manchester`, `liverpool`, `barcelona`, `wnba`, `mls`) return ≥ 1 result.
- Teams that appear across multiple league catalogs (e.g. Arsenal in both Premier League and FA Cup) are de-duped to a single result.
- `lib/sportsdb/` is deleted in its entirety. `grep -rE "thesportsdb|lib/sportsdb"` over `**/*.ts`, `**/*.tsx`, `**/*.json` returns zero hits outside `docs/`.
- `Sport` no longer admits `"Tennis"`. `SPORT_ALLOWLIST` keys are ESPN league keys. All affected test fixtures were migrated to ESPN id shapes.
- All 241 tests pass; lint, typecheck, format all clean.
