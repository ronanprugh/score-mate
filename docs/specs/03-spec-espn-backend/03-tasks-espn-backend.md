### Notes

- Tests are colocated next to the code they test (`foo.ts` ↔ `foo.test.ts`), per `AGENTS.md`.
- Use `pnpm test:ci` to verify changes locally; CI runs `lint → format:check → typecheck → test:ci → build`.
- All commits use Conventional Commits and reference the relevant SDD task in the body, e.g. `Related to T1.2 in Spec 03-spec-espn-backend`.
- No new runtime dependencies — all HTTP via `fetch`; `tsx` (already a devDep) runs the catalog-refresh script.
- The provider-neutral types path (`lib/sports/types.ts`) is intentional so a future provider swap doesn't churn import paths again.
